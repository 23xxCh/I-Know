# 新功能开发

## 核心理念

**不做完需求分析，不动手写代码。**

模糊的需求产出模糊的代码。花时间搞清楚要做什么，比写代码更重要。

---

## 工作流程

### 第一步：需求澄清（Grill模式）

在开始之前，必须回答这些问题：

**必须回答的问题：**

| 问题 | 为什么重要 |
|------|-----------|
| 这个功能解决什么问题？ | 理解目标，避免做错 |
| 用户是谁？怎么使用？ | 理解场景，设计合理 |
| 输入是什么？输出是什么？ | 理解边界，设计接口 |
| 边界情况有哪些？ | 理解完整需求 |

**Grill模式话术：**
```
等等，在你开始之前，我需要确认几个问题：

1. 这个功能是给谁用的？
2. 用户会怎么使用它？
3. 输入是什么？输出是什么？
4. 有什么特殊情况需要处理？

这些不回答清楚，做出来的东西可能不是你想要的。
```

### 第二步：设计先行

写出设计文档（可以很短，但要写）：

**设计文档包含：**

```markdown
## 功能名称

### 目标
[一句话描述这个功能要做什么]

### 接口设计
- 输入：[参数、类型]
- 输出：[返回值、类型]

### 数据流
[数据怎么流转的，简单画一下]

### 与现有代码的关系
[需要改动哪些文件，影响哪些功能]
```

**为什么要写设计文档：**
1. 强迫你把模糊的想法变清晰
2. 给未来的自己和其他人看
3. 写代码时有参考，不会跑偏

### 第三步：测试先行（TDD）

**测试先行的原因：**
- 测试是最好的需求文档
- 测试是安全网，让你敢于修改
- 测试验证你真的完成了

**怎么写测试：**

```python
# 第一步：写一个测试描述你要实现的功能
def test_user_can_login():
    # 给定：一个注册用户
    user = create_user("test@example.com", "password123")

    # 当：用户登录
    result = login("test@example.com", "password123")

    # 那么：登录成功
    assert result.success == True
    assert result.user.email == "test@example.com"
```

### 第四步：垂直切片

**不要水平切分！**

```
错误（水平）：
  RED:   test1, test2, test3, test4, test5
  GREEN: impl1, impl2, impl3, impl4, impl5

正确（垂直）：
  RED->GREEN: test1->impl1 ✅
  RED->GREEN: test2->impl2 ✅
  RED->GREEN: test3->impl3 ✅
```

**为什么垂直切片：**
- 每个切片都是完整的闭环
- 立刻知道对不对
- 可以随时停下来，已有的部分都是可用的

### 第五步：验证完成

**功能完成的定义：**

- [ ] 所有测试通过
- [ ] 边界情况处理
- [ ] 代码简洁（没有冗余）
- [ ] 可以给别人演示

**验证话术：**
```
功能开发完成！来验证一下：

✅ 所有测试通过
✅ 边界情况：[列出来]
✅ 代码简洁，没有冗余

你现在可以演示一下：[告诉用户怎么测试]
```

## 常见问题

### Q: 用户说"很简单，直接写吧"

**回应：**
```
好的，我理解你想快速开始。

但让我先花2分钟确认一下需求，避免写错重写。

[问2-3个关键问题]
```

### Q: 用户说"先写代码，测试以后补"

**回应：**
```
我理解你想快速看到效果。

但测试不是额外的工作，而是开发的一部分：
- 写测试能帮你理清需求
- 没有测试，你不知道代码对不对
- "以后补"的测试通常永远不会写

我建议我们先写测试，这样反而更快，因为不用反复调试。
```

### Q: 需求很复杂，怎么开始

**回应：**
```
这个功能比较复杂，我们拆分一下：

1. [子功能1] - 核心
2. [子功能2] - 扩展
3. [子功能3] - 扩展

我们先从[子功能1]开始，这个是核心，其他的可以后面加。

这样可以吗？
```

## 给新手的话

开发新功能就像盖房子：

1. **需求澄清** = 确认要盖什么房子（几室几厅？多大？）
2. **设计先行** = 画图纸
3. **测试先行** = 打地基（地基不稳，房子会倒）
4. **垂直切片** = 一个房间一个房间地盖，每个房间盖好都能用
5. **验证完成** = 验收，每个房间都检查一遍

不要着急动手。想清楚再写，比写了再改快得多。
