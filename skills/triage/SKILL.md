# Issue 分类（Triage）

## 核心理念

**不是所有 Issue 都需要立刻处理。**

分类的目的：
- 确保重要的问题被优先处理
- 确保 AFK agent 能处理的 issue 不占用人力
- 确保信息不足的 issue 能得到补充

---

## 分类状态

### 状态流转图

```
新 Issue
    ↓
┌─────────────────┐
│  needs-triage   │  需要评估
└────────┬────────┘
         ↓
    ┌────┴────┐
    ↓         ↓
┌───────┐ ┌───────────┐
│wontfix│ │needs-info │ 等待更多信息
└───────┘ └─────┬─────┘
              ↓
         ┌────┴────┐
         ↓         ↓
  ┌────────────┐ ┌──────────────┐
  │ready-for   │ │ready-for     │
  │  -agent    │ │  -human      │
  └────────────┘ └──────────────┘
```

### 状态说明

| 状态 | 含义 | 下一步 |
|------|------|--------|
| `needs-triage` | 新 Issue，需要评估 | 维护者审核 |
| `needs-info` | 信息不足 | 等待用户补充 |
| `ready-for-agent` | 信息完整，AFK agent 可处理 | 自动执行 |
| `ready-for-human` | 需要人工处理 | 分配给开发者 |
| `wontfix` | 不处理 | 关闭 |

---

## 分类标准

### needs-info（需要更多信息）

**触发条件：**
- 没有复现步骤
- 描述模糊（"不工作了"）
- 缺少环境信息
- 截图/日志缺失

**回复模板：**
```
感谢反馈！但我需要更多信息来定位问题：

1. 复现步骤：你是怎么触发这个问题的？
2. 预期结果：你期望发生什么？
3. 实际结果：实际发生了什么？
4. 环境：浏览器版本？操作系统？

请补充这些信息，我会继续跟进。
```

### ready-for-agent（AFK 可处理）

**必须满足的条件：**
- [ ] 需求/问题描述清晰
- [ ] 有明确的输入输出
- [ ] 不需要主观判断
- [ ] 不需要外部协调
- [ ] 技术方案明确

**例子：**
```
标题：添加用户头像上传功能

需求：
- 用户可以上传头像图片
- 支持 JPG、PNG 格式
- 最大 2MB
- 自动裁剪为 200x200

技术方案：
- 使用现有的图片处理库
- 存储到 S3
- 返回图片 URL

这是 ready-for-agent，因为：
- 需求明确
- 技术方案明确
- 不需要主观判断
```

### ready-for-human（需要人工处理）

**触发条件：**
- 需要产品设计决策
- 需要权衡取舍
- 涉及安全/合规
- 需要外部协调
- 技术方案不明确

**例子：**
```
标题：重构用户认证系统

问题：
- 当前认证系统代码混乱
- 有安全隐患
- 扩展性差

这是 ready-for-human，因为：
- 涉及架构决策
- 有安全风险
- 需要权衡取舍
- 需要详细设计
```

### wontfix（不处理）

**触发条件：**
- 超出项目范围
- 已有替代方案
- 技术上不可行
- 成本大于收益

**回复模板：**
```
感谢反馈！

经过评估，我们决定不处理这个 Issue，原因是：

[具体原因]

如果你有不同看法，欢迎继续讨论。
```

## 分类流程

### 第一步：读取 Issue

```markdown
标题：[Issue 标题]
描述：[Issue 描述]
标签：[现有标签]
```

### 第二步：评估

问自己：

1. **这是 Bug 还是功能请求？**
   - Bug → 检查是否有复现步骤
   - 功能 → 检查需求是否清晰

2. **信息够不够？**
   - 不够 → `needs-info`
   - 够 → 继续

3. **能不能自动处理？**
   - 能 → `ready-for-agent`
   - 不能 → `ready-for-human`

4. **该不该处理？**
   - 不该 → `wontfix`

### 第三步：打标签

根据评估结果打上对应标签。

### 第四步：回复（如需要）

- `needs-info`：请求补充信息
- `wontfix`：解释原因
- 其他：可选回复

---

## 给新手的话

Triage 就像医院分诊：

| 状态 | 类比 |
|------|------|
| needs-triage | 病人刚到，需要分诊 |
| needs-info | 病历不全，需要补充信息 |
| ready-for-agent | 小病，护士可以处理 |
| ready-for-human | 大病，需要医生 |
| wontfix | 不需要治疗/无法治疗 |

分诊的目的不是拒绝病人，而是让资源用在最需要的地方。
